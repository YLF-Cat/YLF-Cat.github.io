让 AI 写出可运行的 IC Lua 脚本

没人会为了作图专门学习 Lua！
现在是伟大的 vibe coding 时代！

本教程使用 codex 作为 AI 助手。
请在通读全文后再实践，不要看一点配一点。

为什么要使用 Lua

  1. 为了更高的性能

    请参考 https://blog.ylfcat.top/2025/06/03/ic_test/ 中的性能差距。
    IC Lua 的运行速度极快

  2. 为了更多的功能

    可以 3D，可以调用加速度器，可以绕过权限访问多人函数，可以网络交互，可以播放视频，可以创建 TMP

  3. 为了更方便的编写

    ic 触发器的编写十分困难，Lua 脚本的编写非常容易。
    ic 触发器的维护成本十分高昂，Lua 脚本的维护成本非常低廉。

  4. 为了调用开源库

    例如调用 break_infinity.lua

  5. 为了使用 AI 辅助作图

    减少大量劳动

开始配置

准备

  • 一台电脑
  • VS Code
  • Lua 版本的 ic

VS Code 的配置

  1. 到 https://code.visualstudio.com/Download 下载你操作系统的 VSCode

  2. 点击左侧栏的扩展图标（四个正方形），搜索 Chinese，选择 Chinese (Simplified) (简体中文) 点击 Install

  3. 重启 VSCode

你需要启动 ic，打开你的地图，如果选择添加视觉也需要启动 OBS 虚拟摄像机。

开始愉快的 Vibe Coding

建议

明确向 Codex 提出你要实现什么 Lua 函数，且明确参数。
建议用一张空白地图完成你要实现的函数编写，并将它打包成功能包给你的主图使用。
当你命令 Codex 写一个不涉及 loop 的功能包类似物时，请让它把全部功能实现在新的文件中，以便可以方便拆除脚手架。
有时你也可以将你的触发器截图给 Codex 让它翻译成 Lua。
建议使用触发器里的自定义函数封装你的 Lua 函数。

如果你就是想完全用 AI 实现一个地图，请忽略上面一条。
